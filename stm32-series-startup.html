過程分析

預備知識－系統異常是什麼？
Cortex-M3 有 16 個系統異常，當產生某種異常時，Cortex-M3 根據「異常向量表」定位該類異常對應的處理程序（xxx_handler）的入口地址，並跳轉到該異常處理程序。缺省情況下，CM3 認爲該異常向量表位於 0x00000000 處（若啓動模式爲從 FLASH 中啓動，則自動映射到 0x08000000 處），每個向量佔用四個字節。（異常向量表的位置可通過向量表偏移量寄存器（VTOR 0xE000ED08）更改）

以下爲不使用系統庫（勾選 USE MICROLIB 選項）時的啓動過程。（STM32F103XE）

  • 首先：上電，復位動作開始（復位是優先級最高的系統異常）
    • NVIC 將內核保持在復位狀態（NVIC 對其大部分寄存器清零，處理器處於特權級線程模式，使用 MSP 主棧指針）
    • NVIC 將內核從復位狀態釋放
    • 內核爲 MSP 主棧指針設置初始值（查找異常向量表，從 0x00000000 處讀取四字節數據作爲 MSP 主棧指針的初始值（__initial_sp，由編譯器計算得出））
    • 內核爲 PC 寄存器設置初始值（查找異常向量表，從 0x00000004 處讀取四字節數據作爲 PC 程序計數器初始值（Reset_Handler，由編譯器根據啓動文件得出，指向代碼段中的 Reset_Handler 函數））
  • 然後：根據 PC 的初始值找到 Reset_Handler 函數並執行
    • SystemInit
    • __main
      • __scatterload
        • __scatterload_copy : copies the non-root code and data region from their load-time locations to the execute-time regions.
        • __scatterload_zeroinit : initializes the Zero Initialized (ZI) regions to zero.
      • __rt_entry
        • _platform_pre_stackheap_init
        • __user_setup_stackheap
        • _platform_post_stackheap_init
        • __rt_lib_init
          • _fp_init
          • _init_alloc
          • _rand_init
          • _get_lc_collate
          • _get_lc_ctype
          • _get_lc_monetary
          • _get_lc_numeric
          • _get_lc_time
          • _atexit_init
          • _signal_init
          • _fp_trap_init
          • _clock_init
          • _getenv_init
          • _initio
          • _ARM_get_argv
          • _alloca_initialize
          • _ARM_exceptions_init
          • __cpp_initialize__aeabi_
        • _platform_post_lib_init
        • main()
        • exit()

The _platform_* functions are not part of the standard C library. If you define them, then the linker places calls to them in __rt_entry.
—— ARM Compiler C Library Startup and Initialization
